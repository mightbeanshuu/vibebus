# Product

## Register

brand

## Users

Vibe Bus is for developers running multiple AI coding agents in parallel: Codex, Claude Code, Gemini CLI, Antigravity, Grok, Cursor, Aider, Goose, and custom MCP clients. They are usually in a terminal-heavy workflow, moving fast, and need a lightweight shared coordination layer instead of manually copy-pasting status between agents.

## Product Purpose

Vibe Bus gives CLI agents one shared local bus for identity, messages, threads, handoffs, acknowledgements, tasks, decisions, and status. Success means a developer can start several agents, give them a shared mission, and keep the work synchronized with minimal ceremony.

## Brand Personality

Classy, terminal-native, and slightly memey. The voice should feel like a sharp developer tool with taste: confident, compact, playful in naming, never gimmicky in functionality.

## Anti-references

Avoid generic SaaS pastel gradients, average AI-tool landing pages, purple-on-white defaults, glassy card grids, cluttered npm package pages, and joke-first branding that makes the tool feel unserious.

## Design Principles

- Party line, not dashboard bloat: show the shared bus concept immediately.
- Terminal-first credibility: commands and MCP config should be easy to copy.
- Memorable but usable: the name can be playful, the workflow must be precise.
- Show agents talking: make handoffs, acknowledgements, and threads visible.
- Local and inspectable: emphasize file-backed state and no hidden cloud dependency.

## Accessibility & Inclusion

Target WCAG AA contrast, keyboard-accessible copy buttons and links, responsive layouts down to mobile, reduced-motion alternatives for animations, and text that remains readable in dark-mode-heavy developer environments.
